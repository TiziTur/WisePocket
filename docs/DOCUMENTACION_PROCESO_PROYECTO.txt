====================================================================
1) RESUMEN DEL PROYECTO
====================================================================
Klarity es una plataforma web full-stack para gestion de gastos personales.
Incluye paneles por rol (usuario, asesor, admin), OCR de tickets, recomendaciones y autenticacion reforzada.
Arquitectura basada en frontend estatico + backend NestJS REST con JWT.

4) AUTENTICACION Y AUTORIZACION
====================================================================
4.1 Metodo de autenticacion
- JWT Bearer Token.
- Login y registro tradicional via /auth.
- Login social con Google ID Token.
- Verificacion de email para cuentas password.
- Recuperacion de cuenta por token temporal.

4.2 RBAC
- Roles definidos en enum.
- Decorador @Roles.
- RolesGuard en rutas protegidas.

4.3 Flujo general actual
1. Registro crea usuario con emailVerified=false.
2. Backend genera token de verificacion, guarda hash y envia enlace por email.
3. Usuario verifica correo (endpoint verify-email).
4. Login valida credenciales y estado de verificacion.
5. Backend firma JWT y frontend guarda token.
6. Requests posteriores usan Authorization: Bearer <token>.

====================================================================
5) SEGURIDAD IMPLEMENTADA
====================================================================
- Passwords hasheadas con bcryptjs.
- ValidationPipe global con whitelist, transform y forbidNonWhitelisted.
- DTOs de auth endurecidos (tamano y formato, password fuerte).
- CORS configurable por CORS_ORIGIN.
- Helmet activo para headers de seguridad.
- JWT_SECRET configurable por entorno.
- Guardias de autenticacion y roles.
- Rate limiting global con @nestjs/throttler.
- Throttling adicional por endpoint en auth.
- Tokens de verify/reset almacenados hasheados (SHA-256).
- Normalizacion de email en backend.
- Envio de email tolerante a fallos SMTP (timeout/falla no bloquea flujo de registro o recuperacion).
